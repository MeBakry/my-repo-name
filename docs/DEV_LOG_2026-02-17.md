### 4. Rule Engine (`server/src/services/ruleEngine.js`)

Extracted from the PoC `aiService.js` demo mode into a standalone, production-ready deterministic engine:

- Complexity classification (NAPRA form-to-complexity mapping)
- Frequency/volume assessment with OCP "occasional small quantity" logic
- Risk level determination (A/B/C) from NIOSH table + frequency + batch size
- Exposure risk analysis (skin, eye, inhalation, oral)
- PPE recommendations with rationale
- Facility controls (ventilation, BSC, spill kit, monitoring)
- Full-text rationale generation for each risk level

**No AI. No external API calls. Pure rules + stored data.**

## Architecture Decisions

### Offline-First Data Strategy
All regulatory and chemical data is stored locally as JSON files, seeded into PostgreSQL. No runtime dependency on external APIs. Data updates happen by:
1. Running automated fetcher scripts (`npm run fetch:all`)
2. Running `npm run db:seed` to load into PostgreSQL

### Rule Engine vs AI
The PoC proved that deterministic rules produce correct assessments. The rule engine replaces AI at runtime. AI may be used offline to refine templates in the future, but the production system runs on rules + stored hazard data.

### Frontend Backward Compatibility
The frontend still works standalone (demo mode) when the backend is unavailable. It checks backend health on load and falls back gracefully.

## 8. Automated Data Fetchers (`server/src/data/fetchers/`)

Built automated scripts to fetch real data from public APIs and government sources, replacing manually compiled JSON.

### Scripts

| Script | Source | Method | Output |
|--------|--------|--------|--------|
| `fetch-niosh.js` | CDC/NIOSH 2024 PDF | Downloads PDF from CDC, extracts Table 1 + Table 2 via pdfjs-dist | `niosh-hazardous-drugs.json` |
| `fetch-pubchem.js` | PubChem (NIH/NLM) | PUG REST API for properties + PUG View for GHS hazard classification | `pubchem-data.json` |
| `fetch-health-canada.js` | Health Canada DPD API | Bulk download of 57,788 drug products + 120,153 active ingredient records | `health-canada-dpd.json` |
| `merge-into-ingredients.js` | Local | Merges PubChem + Health Canada data into `ingredients-compounding.json` | Enriched ingredients |

### npm scripts

```bash
npm run fetch:niosh          # Scrape NIOSH PDF
npm run fetch:pubchem        # Fetch from PubChem API (rate-limited ~3 min)
npm run fetch:health-canada  # Download Health Canada DPD (~1 min)
npm run fetch:merge          # Merge all fetched data into ingredients
npm run fetch:all            # Run all fetchers in sequence
```
