Problem We Solve

Compounding pharmacies must maintain a Risk Assessment Record for every preparation they compound. Each record requires:

  • Cross-referencing NIOSH hazardous drug lists
  • Reviewing SDS documents and WHMIS classifications
  • Applying OCP standards and NAPRA complexity guidelines

Without this platform: 45+ minutes per assessment, done by hand.

With this platform: ~15 minutes — most of the work is automated and pre-filled.

Two Core Documents

Master Formulation Record (MFR)

The recipe — product name, ingredients, quantities, form, route, batch size.

Risk Assessment Record

The safety analysis — hazard classification, PPE, controls, risk level (A/B/C).

MFR → triggers → Risk Assessment → pharmacist reviews → supervisor approves → 12‑month review cycle

Pharmacist Flow — Risk Assessment Report

After "Generate Risk Assessment", the system produces a structured report with:

  • Hazard classification per ingredient
  • Complexity level
  • Risk level (A, B, or C)
  • Recommended PPE and facility controls
  • Regulatory references and rationale
📸 Screenshot: Risk Assessment Report Capture: Full report with Level B/C badge, PPE section, Submit for Review button

Workflow: Draft → Submit → Approve

DraftPharmacistGenerates assessment, reviews, can edit
Pending ReviewPharmacistClicks "Submit for Review"
ApprovedSupervisorReviews and approves; sets 12‑month review date

This split ensures independent review for compliance.

When logged in as Supervisor, the screen shows Pending Approvals — assessments submitted for review.

Data & Compliance

  • NIOSH Hazardous Drug List — 213 drugs (Table 1 + 2)
  • Ingredient database — 116 hazard profiles (NIOSH + PubChem + Health Canada)
  • Deterministic rule engine — no AI at runtime; consistent, auditable results
  • Offline-first — hazard data stored locally; no real-time external API calls
